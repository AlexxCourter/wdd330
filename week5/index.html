I thought it was important to take note of how the scope of "this" changes in nested functions. I can forsee this issue being a debugger nightmare otherwise. the "this" keyword binding becomes lost in the scope of a nested function, for example if a forEach loop is used inside a class method. The best way to fix this it appeared was to create a variable equal to "this" in scope of the outer method so that the nested function can use it, or make the nested function an arrow function which will maintain the scope of "this".
