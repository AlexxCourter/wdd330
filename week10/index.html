Week 10 Notes

Questions

  • Is it better to use the audio element in HTML, or in Javascript? Does it matter?

Client side validation should not be the only way that data sent to server is validated. This exposes the server to risks that can be injected into the browser. Javascript validation can easily be turned off by the browser client. Server side validation should follow up client side validation. Client side validation is very useful as a first check that can then prompt the user to enter valid information.

according to this stackoverflow, there is little difference between the javascript Audio object and the HTML audio element. One key function difference is that Javascript code will stop if something like an alert is used, stopping the sound if some other part of the code starts to run. The HTML element audio will play even in this case.
