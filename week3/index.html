Questions

  • Is there a purpose in using function expressions over function declarations, or is it a matter of style?

according to freecodecamp.org, a function declaration has global scope, while a function expression does not. This means that a function declaration is useful when you want to call it from somewhere else in the document and function expressions can be used to maintain clean and compartmentalized code. The sitepoint material on this subject said it is best to be consistent. I think there should be purpose to which is used, and that they should not be interchanged haphazardly.

With the dynamic form exercise, I made a lot of mistakes with javascript functionality that manipulates the DOM (getting HTML elements and changing their state). I was able to solve them, and now know a couple more things to do to ensure that my Javascript code is working. 1 - check the IDs or properties being targeted by a queryselector to make sure they are entered accurately. I spent a significant amount of time staring at an error because I forgot to set an ID on the element I was attempting to target. 2 - the script must be placed at the bottom of the document to access elements above the script call. If the script is called in the head, it cannot access these elements that are below it in the body (at least not when getting the element by ID).
